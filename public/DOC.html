📧 邮件通知服务

基于 Resend 的轻量邮件接口 · 支持纯文本 / HTML · 可携带抄送密送

POST /send | Base URL: https://mailnotify.mnizjc.qzz.io

📖 API 参考文档

🔹 固定发件地址：所有邮件均通过 notification@mnizjc.qzz.io 发送。
参数 from 仅用于自定义显示名称，最终发件人格式为："显示名称 <notification@mnizjc.qzz.io>"。

📬 请求参数

参数 类型 必填 说明
from string 是 发件人显示名称（例如“系统通知”）。
to string / string[] 是 收件人邮箱，支持单个字符串或数组（多个邮箱）。
subject string 是 邮件主题。
text string 是 邮件正文。若 content_type=text/html 则视为HTML。
content_type string 否 text/plain（默认）或 text/html。
cc string / string[] 否 抄送邮箱（格式同 to）。
bcc string / string[] 否 密送邮箱（格式同 to）。

🔐 认证方式

本接口要求使用 Bearer Token 认证。请在请求头中添加：

Authorization: Bearer <your-api-token>

若未提供或 Token 无效，将返回 401 Unauthorized。请联系管理员获取 Token。

✅ 邮箱格式规则

采用严格正则 /^[^\s@]+@([^\s@.,]+\.)+[^\s@.,]{2,}$/
示例合法邮箱: user@example.com , team@company.co.uk
非法: user@.com , name@domain , @missing.com

📮 响应示例

// 成功
{
  "send_id": "abc123-4567-8900-defg-1234567890ab"
}

// 失败（参数错误）
{
  "error": "\"to\" must be valid emails: invalid-email"
}

🔨 cURL 示例

curl -X POST https://mailnotify.mnizjc.qzz.io/send \
  -H "Content-Type: application/json" \
  -d '{
    "from": "运维团队",
    "to": ["admin@example.com"],
    "subject": "部署通知",
    "text": "<h1>Success</h1><p>版本已上线</p>",
    "content_type": "text/html"
  }'

⚠️ 错误码

  • 415 – 不支持的 Content-Type（仅支持 JSON / form-urlencoded / multipart）
  • 200 + error字段 – 业务错误（邮箱格式无效、缺少必填字段）
  • Resend服务错误 – 响应中包含 error.message

※ 接口同时支持 application/json 与表单提交 (urlencoded/multipart)，数组字段使用多个同名参数。
